Что заменено:
- app.py
- complaint_core.py
- browser_bot.py
- templates/complaints.html
- wb_ui_profile.json

Что изменилось:
1) Полностью убрана логика "нет оснований / оснований мало".
2) AI теперь всегда готовит:
   - категорию жалобы
   - короткий текст жалобы до 800 символов
3) Очередь жалоб принимает черновик без проверки "supported/confidence".
4) Браузерный бот открывает отзыв не с общей страницы, а по прямому URL поиска:
   - not-answered?feedbacks-module_searchValue={nmID}
   - answered?feedbacks-module_searchValue={nmID}
5) Поиск нужного отзыва идёт внутри уже отфильтрованной страницы.
6) Если not-answered не сработал, бот автоматически пробует answered.

Как применить:
- замените файлы в проекте
- положите wb_ui_profile.json туда, откуда ваш common.load_ui_profile() его читает
  (если не знаете — сначала просто положите рядом с app.py; если common читает из private dir, скопируйте туда тоже)
- перезапустите сервер: Ctrl+C -> python app.py

Первый тест:
1. В /complaints отметьте 1 отзыв.
2. Нажмите "Подготовить AI-черновики для выбранных".
3. Проверьте категорию и текст.
4. Нажмите "Добавить выбранные черновики в очередь".
5. Поставьте Dry run.
6. Нажмите "Запустить браузерную обработку очереди".

Если бот не найдёт кнопку или окно жалобы:
- пришлите свежий скрин или текст ошибки
- и при необходимости поправим wb_ui_profile.json под ваш интерфейс.
