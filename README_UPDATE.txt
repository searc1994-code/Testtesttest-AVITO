WB Reviews AI + Complaint Queue
================================

Что добавлено
-------------
1. Панель ответов на отзывы: http://localhost:5000/
2. Отдельная панель жалоб на отзывы 1–3★: http://localhost:5000/complaints
3. Вынос чувствительных файлов за пределы проекта:
   - cookies / storage_state
   - локальная очередь жалоб
   - результаты браузерного бота
   - скриншоты попыток отправки
4. Playwright-модуль для жалоб через веб-кабинет WB
5. AI-черновик жалобы: выбирает категорию и пишет короткую причину только если видит достаточные признаки нарушения

Важный принцип
--------------
Система не жалуется автоматически на любой негатив. Жалоба попадает в очередь только если:
- вы сами выбрали отзыв;
- AI увидел достаточные признаки нарушения;
- категория жалобы определена;
- уверенность не ниже порога.

Чувствительные файлы
-------------------
По умолчанию всё чувствительное лежит ВНЕ проекта в папке, указанной в config.py:

    WB_PRIVATE_DIR = r"C:\wb-ai-private"

Там будут файлы:
- auth/wb_state.json
- auth/wb_state_meta.json
- complaints/complaint_drafts.json
- complaints/complaint_queue.json
- complaints/complaint_results.jsonl
- complaints/screenshots/*.png

Если вы архивируете сам проект, эти файлы туда не попадут.

Быстрый запуск
--------------
1. Скопируйте config.example.py в config.py и заполните ключи.
2. Установите зависимости:

    pip install -r requirements.txt
    playwright install

3. Один раз сохраните сессию WB:

    python login_wb.py

4. Запустите веб-панель:

    python app.py

5. Откройте:
   - http://localhost:5000/ — ответы на отзывы
   - http://localhost:5000/complaints — жалобы 1–3★

Как работает жалобная панель
----------------------------
1. Вы открываете /complaints и видите отзывы 1–3★.
2. Ставите галочки у подозрительных.
3. Нажимаете "Подготовить AI-черновики для выбранных".
4. Система показывает:
   - поддерживает ли AI жалобу;
   - категорию;
   - короткую причину;
   - уверенность.
5. Нажимаете "Добавить поддержанные в очередь".
6. Нажимаете "Запустить браузерную обработку очереди" или запускаете:

    python complaint_worker.py --max-items 2

7. Браузер откроется и попробует оформить жалобы через ваш WB Seller кабинет.

Дополнительные проверки
-----------------------
- storage_state лежит вне проекта;
- без сохранённой WB-сессии worker не запустится;
- жалоба не ставится в очередь без достаточной уверенности;
- категория "Отзыв оставили конкуренты" требует повышенной уверенности;
- категория "Другое" тоже требует повышенной уверенности;
- для каждого запуска пишется результат в JSONL и делаются скриншоты.

Что, скорее всего, придётся подстроить вручную
----------------------------------------------
WB может менять DOM в кабинете. Если кнопки жалобы отличаются, подправьте wb_ui_profile.json.
Для точной настройки локаторов удобно один раз посмотреть поведение страницы через Playwright Codegen:

    playwright codegen https://seller.wildberries.ru/reviews

Windows ярлыки
--------------
В папке windows лежат bat-файлы для запуска.
